# gg-vibecode 진입

너(메인 루프)가 대화·구현을 직접 하고, 비대화 작업만 에이전트에 위임한다. 상세는 `CLAUDE.md`.

## 1) 모드·성숙도 판별
- "보안검사만/안전한지" → **검증만**: security-reviewer만 실행 후 종료.
- "배포신청/공식환경 이전" → **배포·이관**: (필요 시 검증 후) deploy-packager.
- "빨리 해줘" → 기본값(직원만·더미·외부통신 없음) 채우고 흐름 확인만.
- 그 외 → **신규**: 아래 흐름.
- 성숙도 기본 L1. 승격 신호(개인정보·시민·외부통신·업로드·지속DB) 있으면 L2/L3 (`maturity-model.md`).

## 2) 신규 흐름 (메인 루프 주도)
1. 접수: "직원만/시민도?" → 존·성숙도. 위험신호 기록. → manifest 초기화.
2. 구상: `socratic-interview`로 화면·행동·입력→출력·MVP 도출 → 되짚기 → `01_요구서.md`.
3. 설계: **solution-architect** 호출 → `02_설계서.md`. 한 줄 통지.
4. 구현: `golden-templates`에서 Track 템플릿 골라 그 안에서 구현(예방 레일).
5. 검증(요청/L3): **security-reviewer** → gvskb. block이면 수정안 받아 4로(최대 2회).
6. 배포·이관(L3): **deploy-packager** → 배포신청·핸드오프.

## 원칙
- 건너뛰기 금지(증거 체인). 위험도 비례(L1 얇게, 🔴 게이트 필수).
- 대민은 사람(위원회) 승인 없이 배포 완료 처리 금지.
- 메인 루프는 gvskb 도구 없음 — 스캔은 security-reviewer만.
- 에이전트 실패 → 1회 재시도 후 "미완료" 명시하고 사용자 판단 요청.
