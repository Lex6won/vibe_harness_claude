# 요구 도출 가이드 (루브릭 + 방법 + 레일)

고정 문항을 읽지 마라. 모두에게 같은 질문 = 룰베이스 챗봇. 대신 아래 **루브릭을 목표**로,
사용자의 말과 만들려는 도구 유형에 맞춰 **질문을 직접 만든다**.

## 메타 규칙
**묻기는 ⓐ사용자가 확실히 알고 ⓑ다른 답에서 파생되지 않는 것만.**
나머지는 파생·기본값·탐지·"신호 있을 때만 확인"으로 조용히 처리한다.

## 3단계

### 1) 추출(Extract)
사용자 프롬프트에서 이미 알 수 있는 걸 먼저 채우고, 루브릭 각 항목을 `[확인됨/부분/빈칸]`으로 표시.
→ **이미 준 건 절대 다시 묻지 않는다.** 상세 브리프면 질문 0개로 바로 되짚기.

### 2) 창의적 질문(Elicit)
`빈칸/부분`인 **기능 항목**만, 도구 유형·사용자 말에 맞춰 질문을 생성.
- 사용자 말을 인용해 이어간다: "민원 *분류*라 하셨는데, 부서별인가요 유형별인가요?"
- 평이·예시·필요시 객관식. 한 번에 쏟지 말고 대화로(답이 다음 질문을 연다).
- 고위험부터. 저위험·불확실 항목은 묻지 말고 기본값.

### 3) 되짚기(Playback)
"제가 이해한 도구는 이거예요: [핵심 화면·기능 요약]. 맞나요?" → 사용자 확인 후 다음 단계.

## 커버리지 루브릭 (항목별 처리 모드)

| 영역 | 알아야 할 것 | 처리 |
|---|---|---|
| 기능 | 핵심 결과물·화면 (표/그래프/폼/검색/지도…) | **묻기(창의)** |
| 기능 | 사용자 행동 (조회/입력/수정/업로드/다운로드/알림) | **묻기(창의)** |
| 기능 | 입력→출력 변환 (출처가 여기서 드러남) | **묻기(창의)** |
| 기능 | 성공기준·꼭 필요한 기능(MVP) | **묻기(창의)** |
| 사용자 | 누가 쓰나 | intake-router가 1회(존/대민). 안 왔으면 여기서 1회 |
| 사용자 | 접근 범위 | 파생 기본값(전체+SSO). red·제한힌트 시만 확인 |
| 데이터 | 출처 | 입력→출력에서 파생. **별도 질문 X** |
| 데이터 | 개인정보·민감도 | **탐지**(설명+코드). 냉질문 금지 |
| 데이터 | 규모·양 | 표준 기본값. 대규모 신호(전 도민/대용량/배치) 시만 거친 확인 |
| 운영 | 지속성(일회성/상시) | 도구 유형에서 추론. 애매 시만 확인 |
| 기술 | Track·인증·스키마 | **정하기**(solution-architect, 안 물음) |

→ **항상 묻는 건 기능 4개 + (필요시)누가 쓰나뿐.** 나머지는 파생/기본값/탐지/신호조건부.

## 멈춤 조건
"N개 물었다"가 아니라 **모든 고위험 항목이 `[확인됨 or 안전기본값]`**이면 멈춘다.
→ 상세 브리프=0질문, 한 줄 요청=여러 질문. 자연스럽게 적응.

## 창의성의 레일 (표류·과다질문·환각 방지)
1. **루브릭 밖으로 새지 않기** — 창의적이어도 커버리지는 보장, 누락 금지.
2. **기술 질문 금지 / 개인정보 등급 냉질문 금지** — 원칙 고정.
3. **예산: 고위험부터, 저위험·불확실은 묻지 말고 기본값** — 피로 방지.

## 조건부 확인 예시 (신호 있을 때만)
- 기존 시스템 연동 암시·불명확 → "그 자료는 기존 시스템/파일에 있나요, 새로 입력받나요?"
- 대규모 신호 → "우리 부서 것 정도인가요, 전 도민 규모인가요?"
- red 데이터·제한 힌트 → "이건 특정 담당자만 봐야 하나요?"

## 확인(Confirm) — 안전기본값 선택된 채 평문
- 존: "직원만 쓰는 걸로 만들게요. 시민도 쓰면 알려주세요."
- 개인정보: "개발·시험은 테스트 데이터로 할게요."
